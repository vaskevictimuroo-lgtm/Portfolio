Sales Statistics API
REST API для анализа статистики продаж, построенное на FastAPI с использованием SQLite в качестве базы данных.

Технологический стек
Backend: FastAPI
База данных: SQLite
Язык: Python 3.x

Функциональность
API предоставляет следующие endpoints:
GET / - Статус приложения
GET /sales - Получить все записи о продажах
GET /total_stats - Получить агрегированную статистику по продажам

Структура данных
Таблица sales содержит следующие поля:
id - уникальный идентификатор (Primary Key, AutoIncrement)
product - название товара
category - категория товара
amount - сумма продажи
date - дата продажи

Установка и запуск
Клонировать репозиторий:

git clone <url-репозитория>
cd <папка-проекта>
Установить зависимости:

pip install fastapi uvicorn sqlite3
Запустить сервер:

uvicorn main:app --reload
После запуска приложение будет доступно по адресу: http://127.0.0.1:8000

Документация API
Автоматически сгенерированная документация доступна по адресу:
http://127.0.0.1:8000/docs - Interactive Swagger UI
http://127.0.0.1:8000/redoc - ReDoc документация

Примеры запросов
Получение всех продаж
bash
curl -X 'GET' 'http://127.0.0.1:8000/sales'
Получение статистики
bash
curl -X 'GET' 'http://127.0.0.1:8000/total_stats'

Архитектура приложения
Приложение использует следующие ключевые компоненты:
FastAPI для создания REST API endpoints
SQLite для хранения данных о продажах
SQLAlchemy-style ручное управление соединениями с БД
Pydantic для автоматической валидации данных и генерации документации

Особенности реализации
Поддержка многопоточности через отдельные соединения с БД для каждого запроса
Автоматическая сериализация данных из SQLite в JSON
Оптимизированные SQL-запросы для агрегации данных

Чистая архитектура с разделением на слои данных и бизнес-логики
